# Farmer Stock Ledger PDF: Summary Table Working

This note explains how the **Summary** table (shown in `FarmerStockLedgerPdf`) is prepared and how every calculated field is derived.

## 1) Entry Point and Data Flow

- `src/components/pdf/FarmerStockLedgerPdf.tsx` is a wrapper.
- It calls `prepareAccountingStockLedgerPdfData(...)` and renders `AccountingStockLedgerPdf`.
- The summary block itself is generated by:
  - `src/components/pdf/summaryTablePdfPrepare.ts`
  - `src/components/pdf/summaryTablePdfCompute.ts`
  - rendered in `src/components/pdf/sumary-table-pdf.tsx`

So, the summary calculations are not implemented in `FarmerStockLedgerPdf.tsx` directly; they come from the summary helpers above.

## 2) How Rows Are Grouped for Summary

`buildGroupedMap(rows)` creates grouped keys:

- Key format: `bagType|size|weightKey|variety`
- Value: total bag count for that exact group

Grouping sources:

- If split bags exist (`sizeBagsJute` / `sizeBagsLeno`), those are used.
- Otherwise legacy/flat fields (`bagType`, `sizeBags`, `sizeWeightPerBag`) are used.

This produces one consolidated quantity per combination of:

- bag type (`JUTE` or `LENO`)
- grading size (for example `Below 30`, `40-45`)
- weight per bag (rounded to `toFixed(2)` as `weightKey`)
- variety

## 3) Summary Row Construction

`getSummaryRows(map)` converts grouped map entries into display rows:

- Fields: `type`, `size`, `count`, `weightNum`, `variety`
- Sort order:
  1. `Below 30` first
  2. then other sizes in grading size order
  3. `JUTE` before `LENO`
  4. lower weight first
  5. variety name

## 4) Per-Row Formula Columns (Right Side)

Computed in `computeSummaryRightValuesForRow(row)`:

Let:

- `count` = number of bags in that row
- `wtPerBag` = row weight (`weightNum`)
- `bardanaPerBag`:
  - `0.7` for `JUTE`
  - `0.06` for `LENO`
- `rate` = buy-back rate from `getBuyBackRate(variety, size)`

Formulas:

1. `WT REC.` (weight received after grading)
   - `wtReceivedAfterGrading = wtPerBag * count`

2. `LESS BARD.`
   - `lessBardanaAfterGrading = count * bardanaPerBag`

3. `ACTUAL WT`
   - `actualWtOfPotato = wtReceivedAfterGrading - lessBardanaAfterGrading`

4. `RATE`
   - Resolved by variety + size from buy-back rate constants

5. `AMT PAY.`
   - `amountPayable = actualWtOfPotato * rate`

## 5) `% of Graded Sizes` Formula

Prepared in `prepareSummaryTablePdfData(...)`.

For each summary row:

- `pctOfGradedSizes = (row.actualWtOfPotato / totalActualWtOfPotato) * 100`

Where:

- `totalActualWtOfPotato` is the sum of `actualWtOfPotato` over all summary rows.

Important:

- This percentage is **weight-based** (actual potato weight), not bag-count-based.
- If `totalActualWtOfPotato <= 0`, the value is shown as `—`.

## 6) Total Row Logic

Totals come from `computeSummaryTotalsFromRows(...)`:

- `totalWtReceivedAfterGrading = Σ row.wtReceivedAfterGrading`
- `totalLessBardanaAfterGrading = Σ row.lessBardanaAfterGrading`
- `totalActualWtOfPotato = Σ row.actualWtOfPotato`
- `totalAmountPayable = Σ row.amountPayable`

And summary percent total:

- `totalPctOfGradedSizes = 100.00%` when total actual weight > 0
- otherwise `—`

## 7) Display Notes

- The summary section is shown only when post-grading bag data is present and grouped rows exist.
- Numeric formatting uses `en-IN` locale.
- `AMT PAY.` and `RATE` are shown with 2 decimals.
- Some zero/invalid cells are displayed as `—` by formatting rules.
